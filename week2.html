Understanding React Hooks

# useState

useState is a React Hook that lets you add state variables to functional components. In the simplest terms:

  • It's like adding memory to your component
  • It helps your component "remember" information between renders
  • When this information changes, React automatically re-renders your component

## Basic Syntax

const [state, setState] = useState(initialValue);

Where:

  • state is the current value
  • setState is a function to update the value
  • initialValue is what you want the state to be when the component first renders
const [email, setEmail] = useState('');

## How it's used:

Example:

// Email input connected to state
<input
    value={email}
    onChange={(e) => setEmail(e.target.value)}
    disabled={isSubmitting}
/>

- email stores what the user types in the input field

- The state affects what's shown in the UI:



Example 2: Dropdown Toggle

In the Dropdown component, useState manages whether the dropdown is open:

const [isOpen, setIsOpen] = useState(false);
  • Controls whether dropdown menu is visible
  • Toggled by a button click
const toggleDropdown = () => {
  setIsOpen(!isOpen);
};

// Button changes appearance based on state
<button onClick={toggleDropdown}>
  <span>{label}</span>
  <svg className={`${isOpen ? 'rotate-180' : ''}`}>...</svg>
</button>

// Dropdown menu only appears when isOpen is true
{isOpen && (
  <div className="dropdown-menu">...</div>
)}

## When to Use useState

You need to remember values between renders

  • Form inputs
  • Toggle states (open/closed)
  • multi-step form
  • ...

UI depends on data that changes

  • Showing/hiding elements
  • Displaying different content based on state
  • Visual feedback (loading states, success/error messages)

User interactions change the component

  • Button clicks
  • Form submissions
  • Keyboard input

## Simple Mental Model

Think of useState as a way to give your component a memory that:

  • Persists between renders
  • Can cause re-renders when updated
  • Can be used to control what appears in the UI

State Management Beyond useState

For more complex applications:

  • useReducer: For complex state logic (similar to Redux)
  • Context API: For sharing state across components without prop drilling
  • State Management Libraries: Redux...

# useEffect

useEffect is a React Hook that lets you perform side effects in your components:

  • It's a way to run code at specific times during your component's lifecycle
  • It handles things that should happen "outside" of the normal rendering process
  • It lets you connect your React component to external systems

## Basic Syntax

useEffect(() => {
  // Code to run after render
  
  return () => {
    // Optional cleanup code (runs before next effect or on unmount)
  };
}, [dependencies]);

Where:

  • The first argument is a function with the code to run
  • The optional return function handles cleanup
  • The dependencies array controls when the effect runs

Example: Detecting Outside Clicks In A Dropdown Menu

In the Dropdown component, useEffect is used to detect clicks outside the dropdown to close it:

useEffect(() => {
  const handleClickOutside = (event: MouseEvent) => {
    if (dropdownRef.current && !dropdownRef.current.contains(event.target as Node)) {
      setIsOpen(false);
    }
  };

  document.addEventListener('mousedown', handleClickOutside);
  return () => {
    document.removeEventListener('mousedown', handleClickOutside);
  };
}, []);

How it works:

  • After the component renders, the effect runs and adds an event listener
  • When a user clicks anywhere, the handler checks if the click was outside the dropdown
  • If it was outside, it closes the dropdown by setting isOpen to false
  • The empty dependency array [] means this effect only runs once after the first render
  • The cleanup function removes the event listener when the component unmounts

## When to Use useEffect

1. For Data Fetching

useEffect(() => {
  const fetchData = async () => {
    setLoading(true);
    try {
      const response = await fetch('https://api.example.com/data');
      const data = await response.json();
      setData(data);
    } catch (error) {
      setError(error);
    } finally {
      setLoading(false);
    }
  };
  
  fetchData();
}, []);  // Empty array means "run once after initial render"

2. For DOM Manipulations

useEffect(() => {
  // Change document title based on count
  document.title = `You clicked ${count} times`;
}, [count]);  // Run when count changes

3. For Timers

useEffect(() => {
  const timer = setInterval(() => {
    setSeconds(seconds => seconds + 1);
  }, 1000);
  
  // Clear interval on unmount
  return () => clearInterval(timer);
}, []);  // Run once on mount

4. For Synchronizing with Props or State

useEffect(() => {
  // Run some code whenever searchTerm changes
  if (searchTerm.length > 3) {
    performSearch(searchTerm);
  }
}, [searchTerm]);  // Run when searchTerm changes

## The Dependency Array Explained

This is the key to controlling when your effect runs:

  • useEffect(() => {}, []) - Runs once after the initial render (similar to componentDidMount)
  • useEffect(() => {}, [value1, value2]) - Runs after initial render and whenever any dependency changes
  • useEffect(() => {}) - Runs after every render

## Mental Model for useEffect

Think of useEffect as a way to synchronize your React component with things outside of React:

  • After React renders your component, it checks if any effects need to run
  • It compares the dependency array with the previous values
  • If any dependencies changed (or if it's the first render), it runs your effect
  • Before running a new effect or unmounting, it runs any cleanup from the previous effect

# useRef

useRef is a React Hook that gives you a persistent reference to a value or DOM element that survives between renders:

  • It's like a "box" that can hold a mutable value
  • The value doesn't trigger re-renders when it changes
  • It's commonly used to access DOM elements directly

## Basic Syntax

const myRef = useRef(initialValue);

Where:

  • myRef is an object with a .current property
  • initialValue is the starting value (often null for DOM refs)
  • You access or modify the value using myRef.current

Example: Tracking DOM Elements

In the Dropdown component, useRef is used to track the dropdown DOM element for detecting outside clicks:

const dropdownRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  const handleClickOutside = (event: MouseEvent) => {
    if (dropdownRef.current && !dropdownRef.current.contains(event.target as Node)) {
      setIsOpen(false);
    }
  };

  document.addEventListener('mousedown', handleClickOutside);
  return () => {
    document.removeEventListener('mousedown', handleClickOutside);
  };
}, []);

// The ref is attached to the DOM element
return (
  <div className="relative mt-1" ref={dropdownRef}>
    {/* dropdown content */}
  </div>
);

How it works:

  • dropdownRef is created with an initial value of null
  • When the component renders, React sets dropdownRef.current to the div element
  • The click handler uses this reference to check if a click was inside or outside the dropdown
  • If the click is outside, it closes the dropdown by setting state

## Key Characteristics of useRef

  • Persistence - The reference survives between renders
  • Mutability - You can change .current without triggering a re-render
  • Synchronous Access - You can read and write to it immediately

## Common Use Cases for useRef

1. Accessing DOM Elements

function TextInputWithFocusButton() {
  const inputRef = useRef(null);
  
  function focusInput() {
    // Directly access DOM methods
    inputRef.current.focus();
  }
  
  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus the input</button>
    </>
  );
}

2. Keeping Track of Previous Values

function Counter() {
  const [count, setCount] = useState(0);
  const prevCountRef = useRef();
  
  useEffect(() => {
    // Store current count in ref after render
    prevCountRef.current = count;
  });
  
  const prevCount = prevCountRef.current;
  
  return (
    <div>
      <p>Current: {count}, Previous: {prevCount}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

## When to Use useRef

You need to access DOM elements directly

  • Focus management
  • Measuring element dimensions
  • Working with third-party DOM libraries

You need to store values between renders that don't affect UI

  • Interval IDs
  • Previous state values
  • Any value that shouldn't trigger re-renders when it changes

You need to detect if something is mounted or not

  • Managing cleanup in async operations
  • Preventing state updates on unmounted components

## A Simple Mental Model

Think of useRef as a persistent "sticky note" attached to your component:

  • You can write and read from it anytime
  • Changing what's written on it doesn't make React redraw your component
  • It remembers what's written on it even when your component re-renders
  • When attached to DOM elements, it gives you a direct reference to those elements

# useLocation

useLocation is a hook from React Router that gives you access to the current URL location in your React component. In simplest terms:

  • It provides information about the current URL the user is viewing
  • It returns a location object with details about the current URL
  • It updates whenever the URL changes, causing components using it to re-render

## Basic Syntax

import { useLocation } from 'react-router-dom';

function MyComponent() {
    const location = useLocation();
    
    // Now you can use location.pathname, location.search, etc.
}

What Location Contains

The location object has these key properties:

  • pathname: The path portion of the URL (e.g., "/products/123")
  • search: The query string including the "?" (e.g., "?sort=price&filter=new")
  • hash: The fragment identifier including the "#" (e.g., "#details")
  • state: Any data that was passed during navigation (e.g., from Link or navigate)
  • key: A unique ID for this location (useful for detecting navigation)

## Common Use Cases for useLocation

1. Highlighting Active Navigation Links

function Navigation() {
    const location = useLocation();
    
    return (
        <nav>
            <ul>
                <li className={location.pathname === "/" ? "active" : ""}>
                    <Link to="/">Home</Link>
                </li>
                <li className={location.pathname === "/about" ? "active" : ""}>
                    <Link to="/about">About</Link>
                </li>
                <li className={location.pathname === "/contact" ? "active" : ""}>
                    <Link to="/contact">Contact</Link>
                </li>
            </ul>
        </nav>
    );
}

2. Parsing Query Parameters

function ProductList() {
    const location = useLocation();
    const queryParams = new URLSearchParams(location.search);
    
    const sort = queryParams.get('sort') || 'default';
    const page = parseInt(queryParams.get('page') || '1', 10);
    
    return (
        <div>
            <h1>Products</h1>
            <p>Sorting by: {sort}</p>
            <p>Page: {page}</p>
            {/* Render products */}
        </div>
    );
}

3. Implementing a "Return to Previous Page" Feature

function ProductDetail() {
    const location = useLocation();
    const navigate = useNavigate();
    
    const returnPath = location.state?.returnTo || '/products';
    
    return (
        <div>
            <h1>Product Detail</h1>
            <button onClick={() => navigate(returnPath)}>
                Back to {returnPath === '/products' ? 'All Products' : 'Previous Page'}
            </button>
        </div>
    );
}

## When to Use useLocation

You need to know the current URL path

  • For highlighting active navigation items
  • For conditional rendering based on the route

You need to access query parameters

  • For filtering, sorting, or pagination
  • For tracking UTM parameters or referrers

You need to access location state

  • For accessing data passed during navigation
  • For implementing flows that preserve context

You need to react to URL changes

  • For analytics tracking
  • For resetting forms or state when paths change

## A Simple Mental Model

Think of useLocation as your "GPS" within your React application:

  • It tells you exactly where the user currently is in your app
  • It provides all the details about the current URL
  • It updates whenever the location changes
  • Components using it will re-render when the location changes

## React Router (Beyond useLocation)

For more complete routing solutions:

  • Route Configuration: Setting up your application's routes
  • Nested Routes: Routes within routes for complex UIs
  • Route Parameters: Capturing dynamic segments of the URL
  • Protected Routes: Restricting access to certain routes

# Creating Components in React

## Functional Component Basics

React components are typically created as JavaScript/TypeScript functions that return JSX. Here's the basic pattern: 

import React from 'react';

function MyComponent() {
  return (
    <div>
      <h1>Hello World</h1>
      <p>This is my first component</p>
    </div>
  );
}

export default MyComponent;

## Component with Props

Props allow you to pass data to your components: 

import React from 'react';

// Using TypeScript to define the props type
interface ButtonProps {
  text: string;
  variant?: 'primary' | 'secondary';
  onClick?: () => void;
}

function Button({ text, variant = 'primary', onClick }: ButtonProps) {
  const buttonClasses = variant === 'primary' 
    ? 'bg-blue-600 text-white' 
    : 'bg-gray-200 text-gray-800';
    
  return (
    <button 
      className={`px-4 py-2 rounded ${buttonClasses}`}
      onClick={onClick}
    >
      {text}
    </button>
  );
}

export default Button;

## Component with State

Components can manage their own state using hooks: 

import React, { useState } from 'react';

function Counter() {
  // Initialize state with a value of 0
  const [count, setCount] = useState(0);
  
  // Functions to update state
  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);
  
  return (
    <div className="counter">
      <h2>Count: {count}</h2>
      <div className="buttons">
        <button onClick={decrement}>-</button>
        <button onClick={increment}>+</button>
      </div>
    </div>
  );
}

export default Counter;

## Component File Structure

Each component file should:

  • Import dependencies
  • Define types/interfaces (if using TypeScript)
  • Define the component function
  • Export the component
// Import dependencies
import React, { useState } from 'react';
import Button from '../ui/Button';

// Define types
interface NewsletterProps {
  title?: string;
}

// Define component
const Newsletter = ({ title = "Subscribe to our newsletter" }: NewsletterProps) => {
  const [email, setEmail] = useState('');
  
  const handleSubmit = (e) => {
    e.preventDefault();
    // Handle the subscription
    console.log(`Subscribing ${email}`);
  };
  
  return (
    <section className="newsletter">
      <h2>{title}</h2>
      <form onSubmit={handleSubmit}>
        <input 
          type="email" 
          value={email} 
          onChange={(e) => setEmail(e.target.value)} 
          placeholder="Your email"
        />
        <Button text="Subscribe" />
      </form>
    </section>
  );
};

// Export component
export default Newsletter;

### Passing Props to Components

Props are passed as attributes in JSX: 

// Basic props
<Button text="Click me" variant="primary" />

// Event handlers
<Button text="Submit" onClick={() => console.log('Button clicked')} />

// Children prop
<Card>
  <h2>Card Title</h2>
  <p>Card content goes here</p>
</Card>

// Spreading props
const buttonProps = {
  text: "Click me",
  variant: "primary",
  onClick: () => alert("Clicked!")
};
<Button {...buttonProps} />

### Component Composition Patterns

1. Children Props

Components can accept content through the children prop: 

// Defining the component
function Card({ title, children }) {
  return (
    <div className="card">
      {title && <h3 className="card-title">{title}</h3>}
      <div className="card-content">
        {children}
      </div>
    </div>
  );
}

// Using the component
<Card title="Important Info">
  <p>This is some important information.</p>
  <button>Learn More</button>
</Card>

2. Component Composition

Break complex UIs into smaller, focused components: 

function ProductPage() {
  return (
    <Layout>
      <ProductHeader />
      <div className="content">
        <ProductGallery />
        <ProductDetails />
      </div>
      <RelatedProducts />
    </Layout>
  );
}

## Best Practices for Creating Components

Single Responsibility Principle

  • Each component should do one thing well
  • If a component gets too big, break it into smaller components

Keep Components Pure When Possible

  • Given the same props, a component should always render the same output
  • Isolate side effects to specific components or hooks

Use Descriptive Naming

  • Component names should clearly describe what they do
  • Use PascalCase for component names (e.g., UserProfile, not userProfile)

Prop Validation

  • Use TypeScript interfaces or PropTypes to define props
  • Provide default values for optional props

Consistent File Structure

  • One component per file (usually)
  • File name should match the component name

Custom Hooks for Reusable Logic

  • Extract complex logic into custom hooks
  • Prefix custom hook names with "use" (e.g., useFormValidation)
