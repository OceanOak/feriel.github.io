Exploring Pusher

Pusher

Pusher is a platform that provides real-time communication APIs and infrastructure, allowing developers to build features like:

  • Live chat
  • Real-time notifications
  • Live dashboards
  • Collaborative apps

It's most known for its Channels and Beams products.

When a publisher sends data to a channel, Pusher uses WebSockets to instantly deliver that data to all subscribers who are listening.

Publisher

The user who sends (or "publishes") an event that others should know about.

Subscriber

The user who listens for and reacts to those events.

Channels

The connection between publishers and subscribers. Publishers send messages to a channel, and subscribers listen to that channel to get updates.

Channels

Channels uses WebSockets to push data from your server to clients instantly — without the client needing to constantly poll for updates.

It's a hosted Pub/Sub messaging API that allows you to send and receive messages in real-time.

  • ➤

    WebSocket-based messaging: Once a client connects, the server can push updates in real-time.

  • ➤

    Hosted Pub/Sub messaging API: Pusher handles all the server infrastructure so you don't have to.

  • ➤

    For in-app real-time: Getting info instantly, no refresh (seeing who's online, etc.)

  • ➤

    Persistent connections: Uses WebSockets and HTTP to allow for persistent two-way connection between client and server.

Channel Types

Public Channels

Anyone can see, no restrictions

Private Channels

Authentication required

Presence Channels

Authentication required, you can see who is online

Channels Dashboard

The Channels dashboard gives you a live look at what is happening: connections, messages flowing, who is subscribed, and more.

Realtime activity is shown in your Channels dashboard and can be sent to your system via webhooks.

Channels is used for realtime charts, realtime user lists, realtime maps, multiplayer gaming, and many other kinds of UI updates.

Beams

Beams handles push notifications and makes sending them easy.

It's designed for out-of-app notifications, sending push notifications across various platforms. Pusher provides SDKs to make integration simple.

Delivery Methods

Authenticated Users

Send notifications just to specific users (direct message notifications, etc.)

Device Interests

Users subscribe to channels or topics (everyone subscribed to a channel will get the notification)

Common use cases include tracking delivery drivers and transaction notifications from banks.

Additional Features

  • Hosted token management:

    When a user installs your app, their device gets a unique push token from Google or Apple (essentially an address for notifications). These tokens can expire or change. Beams keeps the tokens fresh by communicating with platform gateways.

  • Webhooks:

    Let developers' own servers react to notifications, providing realtime observability into the notification service (from device registration to notification opened).

  • Insights:

    Monitor delivery rates, open rates, and more.

  • Debug console:

    Offers complete visibility into your notification service, tracking everything from initial device registration to when users open notifications.

Key Takeaways

  • ✓

    Scalable, easy to integrate: Pusher provides hosted APIs that handle the complexity of real-time messaging and push notifications.

  • ✓

    Two main products: Channels for in-app real-time updates, and Beams for push notifications.

  • ✓

    Publish-Subscribe model: A simple pattern where publishers send events through channels, and subscribers receive those events.

  • ✓

    Developer-friendly: Dashboard tools help monitor your real-time applications and troubleshoot issues.
